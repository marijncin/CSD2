Antwoord 1: 
git diff laat de veranderingen (changes) zien tussen twee versies van bestanden. 
Daarnaast laat git diff ook de verschillen zien tussen 2 branches. 

Antwoord 2:
git fetch wordt gebruikt wanneer je op andermans repository aan het werk bent en een bestand wil downloaden naar je eigen local disk. git fetch overschrijft niet je locale bestand. dit doet git pull wel. Als je je locale bestand wil vervangen moet je een git merge uitvoeren. 

Antwoord 3:
git pull: 
Haalt data binnen van de GitHub repository en download het naar je systeem, waar het de oude kopie overschrijft. Het dient als een soort refresh functie. 

Antwoord 4: 
Git log: 
git log laat een logboek zien van je git commit geschiedenis. 
je kunt ook git log --oneline gebruiken, dit maakt alles overzichtelijker. 

Antwoord 5: 
git push:
git push stuurt de bestanden die lokaal gecommit zijn naar git door naar je gitgub repository. 

Git justification:
1: Versiebeheer van bestanden blijft d.m.v branches overzichtelijk
2: Bestanden die je op je pc kwijtraakt kunnen worden hersteld vanuit de cloud
3: Samenwerken aan dezelfde code of hetzelfde project wordt een stuk gemakkelijker.
